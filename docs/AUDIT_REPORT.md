## 三、下一步开发计划

### 3.1 Sprint 1: 核心功能完善 (预计 2 周)

**目标**: 完成核心 API 和用户验证流程

#### P0 - 必须完成

1. **完善邮箱验证 API**
   - [ ] 实现验证码生成和存储 (`internal/core/email/verification.go`)
   - [ ] 完善 `/api/v1/user/verify-email` 端点
   - [ ] 添加验证码过期机制 (30分钟)

2. **完善 API 认证中间件**
   - [ ] 实现签名验证逻辑
   - [ ] 添加时间戳防重放检查
   - [ ] 从请求中提取用户信息

3. **完善条目 Handler**
   - [ ] `/api/v1/entry` POST 创建条目 - 添加签名验证
   - [ ] `/api/v1/entry/{id}` PUT 更新条目 - 权限检查
   - [ ] `/api/v1/entry/{id}` DELETE 删除条目 - 软删除实现

4. **完善用户 Handler**
   - [ ] `/api/v1/user/register` POST - 完整注册流程
   - [ ] `/api/v1/user/info` GET - 返回用户详情
   - [ ] `/api/v1/user/info` PUT - 更新用户信息

#### P1 - 重要功能

5. **用户层级升级定时任务**
   - [ ] 实现 `CheckLevelUpgrade` 定时任务 (每小时)
   - [ ] 在评分/贡献时触发升级检查

6. **初始分类初始化**
   - [ ] 创建初始分类数据 (`configs/seed-categories.json`)
   - [ ] 节点启动时自动初始化分类

7. **种子节点初始数据**
   - [ ] 创建默认知识库数据 (`configs/seed-data/`)
   - [ ] 种子节点启动时自动导入

### 3.2 Sprint 2: 网络功能增强 (预计 2 周)
